# LogMaker: 제품 요구사항 문서 (PRD)

**작성자:** Gemini
**버전:** 2.0
**최종 수정일:** 2026년 3월 19일

---

## 1. 개요 (Overview)

### 1.1. 문제점 (Problem)
소프트웨어 개발, 테스트, 보안 분석 과정에서 특정 형식과 내용을 갖춘 대규모 로그 데이터를 생성하고 검증하는 것은 매우 중요합니다. 하지만 수동으로 이러한 데이터를 만들거나 기존 시스템의 로그를 원하는 형태로 가공하는 것은 비효율적이고 시간이 많이 소요되는 작업입니다. 특히, 다양한 로그 포맷(예: 웹 서버 로그, 시스템 로그, 애플리케이션 트랜잭션 로그)과 전송 프로토콜(예: Kafka, Syslog)을 동시에 처리해야 할 경우 복잡성은 더욱 증가합니다.

### 1.2. 해결책 (Solution)
**LogMaker**는 사용자가 웹 UI를 통해 직관적으로 로그 데이터의 구조를 정의하고, 원하는 대상으로 실시간으로 로그를 전송할 수 있게 해주는 유연한 로그 생성 및 전송 플랫폼입니다. 플러그인 기반 아키텍처를 채택하여 새로운 종류의 데이터 생성기(Maker)나 전송기(Sender)를 손쉽게 확장할 수 있습니다.

### 1.3. 목표 (Goals)
- **직관적인 로그 생성:** 사용자가 코딩 없이 UI를 통해 다양한 데이터 타입(IP, 날짜, 정규식 기반 문자열 등)을 조합하여 복잡한 로그 메시지 포맷을 정의할 수 있도록 합니다.
- **유연한 확장성:** 개발자가 새로운 데이터 생성 방식(Maker)이나 전송 대상(Sender)을 플러그인 형태로 손쉽게 개발하고 시스템에 추가할 수 있는 환경을 제공합니다.
- **실시간 모니터링:** 생성되는 로그의 처리량(TPS)과 상태를 대시보드에서 실시간으로 확인하고 제어할 수 있는 기능을 제공합니다.
- **다양한 전송 대상 지원:** 주요 로그 수집 시스템(Kafka, Syslog 등)에 대한 기본 전송 플러그인을 제공하여 별도의 설정 없이 즉시 연동할 수 있도록 합니다.

## 2. 사용자 페르소나 (User Personas)

- **소프트웨어 개발자:** 애플리케이션의 로그 처리 로직을 테스트하기 위해 특정 형식의 로그 데이터가 대량으로 필요한 개발자.
- **QA 엔지니어:** 시스템의 부하 테스트나 통합 테스트 시, 실제와 유사한 트래픽 패턴을 가진 로그를 생성하여 시스템의 안정성을 검증하려는 엔지니어.
- **보안 분석가:** 특정 공격 패턴(예: SQL Injection, XSS)이 포함된 로그를 생성하여 보안 장비(SIEM, WAF)의 탐지 룰을 테스트하고 검증하려는 분석가.
- **DevOps/SRE:** 새로운 모니터링 또는 얼럿 시스템을 구축할 때, 다양한 로그 소스를 시뮬레이션하여 시스템이 정상 동작하는지 확인하려는 엔지니어.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 핵심 기능: 로그 생성 및 관리 (Log Generation & Management)
- **로그 템플릿 생성:** 사용자는 UI에서 텍스트와 여러 'Maker' 컴포넌트를 조합하여 로그 메시지 템플릿을 생성할 수 있다.
  - 예: `{ip} - - [{timestamp}] "GET /api/users HTTP/1.1" 200 {response_time}`
- **로그 생성 제어:**
  - 초당 생성 개수(TPS)를 지정하여 로그 생성 속도를 조절할 수 있다.
  - 로그 생성을 시작/중지/재시작할 수 있다.
- **실행 상태 확인:** 현재 실행 중인 로그 생성 작업 목록과 각각의 상태(실행 중, 중지됨), TPS, 총 생성 개수를 확인할 수 있다.

### 3.2. 데이터 생성기 (Makers)
- 시스템은 다양한 종류의 데이터를 생성하는 'Maker' 플러그인을 기본으로 제공한다.
- **기본 제공 Makers:**
  - `DateMaker`: 지정된 포맷의 현재 시간 또는 특정 시간대의 날짜/시간 문자열을 생성한다.
  - `IPMaker`: IPv4 주소를 생성한다. (특정 대역 지정 가능)
  - `NumberRangeMaker`: 지정된 범위 내의 정수 또는 실수를 랜덤하게 생성한다.
  - `PickMaker`: 사용자가 입력한 여러 문자열 중 하나를 랜덤하게 선택한다.
  - `RegexMaker`: 주어진 정규 표현식에 맞는 문자열을 생성한다.
  - `UUIDMaker`: UUID를 생성한다.

### 3.3. 로그 전송기 (Senders)
- 생성된 로그를 외부 시스템으로 전송하는 'Sender' 플러그인을 제공한다.
- 사용자는 생성된 로그를 하나 이상의 Sender로 동시에 전송하도록 설정할 수 있다.
- **기본 제공 Senders:**
  - `DebugSender`: 로그를 화면(UI) 또는 서버 콘솔에 출력한다. (테스트 및 디버깅용)
  - `KafkaSender`: 지정된 Kafka 토픽으로 로그를 전송한다. (Broker 주소, Topic 지정 필요)
  - `SyslogSender`: 지정된 Syslog 서버로 로그를 전송한다. (UDP/TCP, Host, Port 지정 필요)

### 3.4. 플러그인 시스템 (Plugin System)
- **플러그인 관리:**
  - UI를 통해 현재 설치된 Maker 및 Sender 플러그인 목록을 확인할 수 있다.
  - 새로운 플러그인(`*.jar` 파일)을 시스템에 업로드하여 동적으로 기능을 확장할 수 있다.
- **플러그인 API:**
  - `plugin-api` 모듈은 `MakerPlugin`과 `SenderPlugin` 인터페이스를 제공한다.
  - 개발자는 이 인터페이스를 구현하여 자신만의 Maker와 Sender를 개발할 수 있다.

### 3.5. 웹 UI (Web User Interface)
- **기술 스택:** SvelteKit + Svelte 5 (runes), Tailwind CSS v4, TypeScript
- **대시보드:** 전체 로그 생성 현황(총 EPS, 실행 중인 작업 수, CPU/메모리 사용률)을 실시간으로 보여준다. (5초 간격 자동 갱신)
- **Maker/Sender 관리 페이지:** 사용 가능한 Maker와 Sender 목록 및 각각의 설정 옵션을 보여준다. Import/Export, DynamicInput 지원.
- **로그 생성 페이지:** 로그 템플릿을 정의하고(커서 위치에 Maker 삽입), 사용할 Sender를 선택하며, EPS 등을 설정하여 새로운 로그 생성 작업을 시작하는 페이지. 실시간 미리보기 지원.
- **플러그인 관리 페이지:** 설치된 플러그인을 확인하고 드래그앤드롭으로 새로운 플러그인을 업로드하는 페이지.
- **반응형 디자인:** 데스크탑/태블릿/모바일 환경 지원, 다크 모드 내장.

## 4. 비기능적 요구사항 (Non-Functional Requirements)

- **성능 (Performance):** 단일 인스턴스에서 최소 10,000 TPS 이상의 로그 생성이 가능해야 한다.
- **확장성 (Scalability):** 플러그인 아키텍처를 통해 기능 확장이 용이해야 한다. 추후 여러 인스턴스로 수평 확장이 가능하도록 고려한다.
- **사용성 (Usability):** 사용자가 매뉴얼 없이도 주요 기능을 쉽게 이해하고 사용할 수 있는 직관적인 UI를 제공해야 한다.
- **안정성 (Stability):** 특정 플러그인에서 오류가 발생하더라도 전체 시스템의 안정성에 영향을 주지 않아야 한다.

---

## 5. 범위 외 (Out of Scope)

- **사용자 인증/인가:** 현재 버전에서는 별도의 사용자 계정 관리나 권한 제어 기능을 포함하지 않는다. (단, Actuator 엔드포인트 접근은 health/info만으로 제한됨)
- **생성된 로그의 저장 및 분석:** LogMaker는 로그를 생성하고 전송하는 역할만 수행하며, 생성된 로그를 자체적으로 저장하거나 분석하는 기능은 제공하지 않는다.
- **플러그인 마켓플레이스:** 플러그인을 공유하거나 검색할 수 있는 중앙화된 저장소는 구축하지 않는다.
- **데이터베이스:** 설정 데이터는 JSON 파일 기반으로 영속화된다. (ACID가 필요한 경우 향후 임베디드 DB 전환 검토)

## 6. 향후 계획 (Future Considerations)

- **고급 Maker/Sender 추가:**
  - AWS S3, Google Cloud Storage 등 클라우드 스토리지로 전송하는 Sender.
  - 데이터베이스(JDBC)로 전송하는 Sender.
  - 파일에서 데이터를 읽어와 사용하는 `FileBasedMaker`.
- **시나리오 기반 로그 생성:** 시간의 흐름에 따라 TPS나 로그 패턴이 동적으로 변경되는 시나리오 기능.
- **Web-hook 지원:** 로그 생성 시작/중지 등 특정 이벤트 발생 시 지정된 URL을 호출하는 기능.
